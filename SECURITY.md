# Security Policy

## Supported Versions

Security fixes are currently provided for the latest version on the `main` branch.

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Report security issues privately with the following details:
- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Suggested fix (if available)

Preferred contact methods:
- Email: muhammadasifkha01@gmail.com
- Or open a private security advisory in GitHub (if enabled)

## Response Targets

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Regular status updates: at least once per week until resolved

## Disclosure Process

- We follow coordinated disclosure.
- A fix will be prepared and released before full public disclosure whenever possible.
- Reporters will be credited unless they request anonymity.
